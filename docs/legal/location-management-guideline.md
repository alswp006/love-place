# love place 위치정보 취급·관리 지침 (내부)

> 「위치정보의 보호 및 이용 등에 관한 법률」 시행령 제20조에 따른 **내부 관리용 지침**.
> 공개 게시하는 `location-policy.md`(위치정보처리방침)와 **별개 문서**다. 이 지침은 사업자 내부 운영 기준이며,
> 위치기반서비스사업 신고 시 보호조치 증빙으로 제출/보관한다. 출시 전 `[ ]` 항목을 채운다.

## 1. 관리책임자
- 위치정보관리책임자: `[성명/직위]` · 연락처: `[이메일/전화]`
- 책임: 동의·파기·접근통제·확인자료 관리, 정기 자체점검, 침해사고 대응.

## 2. 처리 대상·목적·보유
- 대상: 개인위치정보(이용자 단말 GPS 동선) — '여행 시작~종료' 세션 중에 한해 수집.
- 목적: 연결된 두 사람의 여행 동선 기록·리캡.
- 보유: 동선은 목적 달성·삭제 요청·동의 철회 시 지체 없이 파기. **위치정보 이용·제공사실 확인자료는 6개월 보존 후 자동 파기**(동의 철회 시 동선과 함께 즉시 파기 — 제24조4).

## 3. 동의 관리 (제18·19조)
- 4종 동의를 분리·기본 OFF로 받고 `consent_log`(append-only)에 기록(유형·범위·정책버전·표시문구 해시·시각).
- 수집·이용 동의(COLLECT_USE) 없이는 서버 RPC(`record_points`)가 점을 거부.
- 제3자(상대) 제공은 별도 동의(THIRD_PARTY_PROVIDE_PARTNER) — 미동의 시 서버 RPC(`get_session_points`)가 좌표 반환 거부.
- 동의 철회·일시중지는 '우리' 탭 컨트롤 센터에서 항상 가능(거절 불가 — 제24조2).

## 4. 안전성 확보 조치 (제16조)
- **암호화**: 좌표(`route_points`)는 pgcrypto 컬럼 암호화(at-rest) + 전송 TLS. 키는 Vault 시크릿(서버 함수만 복호).
- **접근통제**: RLS 커플 단위 격리(`current_couple_id()`). 좌표 복호 read/write는 SECURITY DEFINER RPC만(직접 테이블 접근 GRANT 없음).
- **접근기록**: 수집·이용·제공사실 확인자료(`location_access_log`, append-only)를 자동 기록. Supabase 접속로그 별도 보존.
- **최소수집**: distanceFilter 간헐 샘플링, 세션 한정, 상시추적 금지.

## 5. 위치정보 이용·제공사실 확인자료 (취급대장 역할 — 제16조2)
`location_access_log`가 법정 확인자료(취급대장)를 시스템으로 대체한다. 기록 항목:
| 항목 | 컬럼 |
|---|---|
| 처리 유형(수집/이용/제공) | `event_type` |
| 처리 일시 | `event_at` |
| 개인위치정보주체 | `data_subject_id` |
| 취급자/요청자 | `actor_id` |
| 제공받은 자 | `recipient_id` |
| 목적 | `purpose` |
| 보존기한 | `retain_until`(=event_at+6개월) |

- 변경·삭제 불가(append-only, RLS에 UPDATE/DELETE 정책 없음). 만료분은 `purge_expired_access_log` 잡이 자동 파기.

## 6. 파기 (제23·24조)
- 평시: `purge_expired_access_log`(pg_cron 일 1회)가 보존기간 경과 확인자료 파기. 휴지통 동선은 유예 경과 후 정리.
- 철회: `purge_location_data`(service_role, location-purge Edge Function 경유)가 해당 세션의 동선 + 확인자료를 **복구불가 하드 DELETE**. 클라이언트 캐시도 evict.

## 7. 정기 자체점검 (제16조)
- 분기 1회: 접근권한 적정성, 확인자료 기록·파기 동작, 동의 철회 파기 동작, 암호화 키 관리 점검. 결과 `[기록 위치]`에 보관.

## 8. 침해사고 대응
- 유출·오접근 인지 시 즉시 차단·원인분석, 필요 시 KISA/방통위 신고 및 이용자 통지.

> **시행일: `[시행일]`** · 개정 시 본 지침과 공개 처리방침을 함께 갱신한다.
