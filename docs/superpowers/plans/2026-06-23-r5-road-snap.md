# R5 도로 스냅 — 측지선 베이스 + 스냅 오버레이 Implementation Plan

> 근거: 심층 검토(wallHeight=low, 진짜 블로커 0). 기존 프록시 인프라(_shared/middleware·ai-route 템플릿) 위 얇은 어댑터.
> 원칙: **측지선=즉시 그리는 베이스라인, 도로 스냅=비동기로 덮는 프로그레시브 인핸스먼트.** 함수 미배포/실패/오프라인이면 측지선 유지(안 죽음).

## 조각
### S1 — directions Edge Function (`supabase/functions/directions/index.ts`)
- ai-route/index.ts 템플릿 복제: authenticate→checkRateLimit(FN='directions', PER_MIN=20, PER_DAY=200)→월상한 가드(MONTHLY_CAP_DIRECTIONS, 기본 넉넉)→leg별 캐시→외부호출→recordCall.
- 입력 `{ legs: [{from:{lat,lng}, to:{lat,lng}}], mode? }`. **leg = 인접 N-1 구간(경유지 없음)** → provider 경유지 상한 무관(03-proxy-contract §b).
- leg별: cache_key `directions:` + sha256(`CAR|kakao|{from5}|{to5}`)(좌표 5자리), TTL 24h. 캐시 히트면 재사용.
- 외부: 1차 카카오모빌리티 `GET apis-navi.kakaomobility.com/v1/directions?origin=lng,lat&destination=lng,lat`(헤더 `Authorization: KakaoAK <KAKAO_REST_KEY>`) → routes[0].sections[].roads[].vertexes([lng,lat] 평탄) → {lat,lng}[] + summary.distance(m). 실패 시 2차 TMap `POST apis.openapi.sk.com/tmap/routes`(appKey: TMAP_APP_KEY) → GeoJSON LineString. 둘 다 실패 → degraded(폴리라인 없음). **degraded leg는 캐시 금지**(일시실패 고정 방지).
- 출력 `{ ok:true, legs:[{polyline:[{lat,lng}]|null, distanceMeters:number|null, degraded:bool}] }`.
- **배포·시크릿은 사용자 단계**(`supabase functions deploy directions`, KAKAO_REST_KEY 확인/TMAP_APP_KEY 설정). 미배포여도 클라가 측지선 폴백.

### S2 — leg 도출(순수) `src/lib/recap/legs.ts` + 테스트
- `toLegs(vertices)`→인접 쌍, `verticesKey(vertices)`(id+좌표+순서 해시, 쿼리키·무효화), `mergeLegPolylines(legResults, legs)`(leg 폴리라인 순서대로 concat; degraded leg는 [from,to] 직선으로 연속성 유지), `roadDistanceKm(legResults)`(distanceMeters 합/1000; 하나라도 null이면 null).

### S3 — `src/hooks/useSnappedPolyline.ts` + 테스트
- TanStack Query `['trip-directions', coupleId, tripId, verticesKey]`, queryFn=`supabase.functions.invoke('directions',{legs})`, enabled=vertices≥2 && coupleId && isSupabaseConfigured, retry:false, staleTime 길게.
- 반환 `{ polyline:{lat,lng}[]|null, roadDistanceKm:number|null, degraded:boolean, isLoading }`. 에러(미배포/네트워크)면 polyline=null(→측지선). useKakaoSearch invoke/에러 패턴 복제.

### S4 — RecapPage 프로그레시브 통합
- 측지선 polyline 즉시 렌더(기존). `useSnappedPolyline` 도착 시 polyline 교체(`snapped.polyline ?? geodesic`). 스탯 거리: `snapped.roadDistanceKm ?? stats.distanceKm`, 라벨 `도로`(스냅) vs `장소→장소`(측지선) — 색+텍스트 이중화. degraded면 측지선 라벨 유지.

### S5 — 게이트 + 푸시 + 인계
- tsc0/vitest(legs·useSnappedPolyline)/build/e2e. Edge Function은 Deno라 앱 tsc 제외(코드 리뷰로 검증). 인계: 함수 배포 + 시크릿 + (카카오모빌리티 쿼터 확인/NCP면 Directions 활성화).
## 정직성
스냅 적용 시 distance 의미가 도로거리로 바뀜 → 라벨 분기로 명시. CAR 전용 멀티모달 왜곡은 degraded→측지선 폴백 + 라벨. Edge Function 실배포·실기기는 사용자 확인.
